## 一、产品概述

### 1.1 产品定位

meodou APP是meodou品牌智能宠物用品的配套移动应用，为用户提供设备远程控制、健康监测、数据分析等一站式宠物管理服务。

### 1.2 目标用户

| 用户画像 | 特征描述 |
|----------|----------|
| **主要用户** | 25-40岁城市养猫人群，追求科技养宠 |
| **次要用户** | 多猫家庭、经常出差的宠物主人 |
| **潜在用户** | 关注宠物健康、愿意尝试智能产品的养宠人群 |

### 1.3 核心价值

- **实时监控**：随时了解宠物用品使用状态
- **健康洞察**：通过数据分析宠物健康状况
- **远程控制**：随时随地操控智能设备
- **智能提醒**：耗材不足、异常行为及时通知

## 三、详细功能规格

### 3.1 首页模块

#### 3.1.1 健康评分卡片

| 属性 | 规格说明 |
|------|----------|
| **功能描述** | 展示宠物综合健康评分 |
| **数据来源** | 综合设备使用数据分析得出 |
| **评分范围** | 0-100分 |
| **更新频率** | 每日更新一次 |
| **交互** | 点击进入健康报告详情页 |
| **显示元素** | 评分数字、较昨日变化、趋势箭头 |

**计算逻辑：**
```
健康评分 = 如厕频率得分(30%) + 如厕时长得分(25%) + 活动量得分(25%) + 休息质量得分(20%)
```

#### 3.1.2 设备状态卡片

| 属性 | 规格说明 |
|------|----------|
| **功能描述** | 展示已绑定设备的实时状态 |
| **显示内容** | 设备图标、名称、在线状态、关键指标 |
| **交互** | 点击进入对应设备详情页 |
| **刷新方式** | 进入页面自动刷新 + 下拉刷新 |

**智能猫厕所显示指标：**

| 指标 | 说明 | 数据来源 |
|------|------|----------|
| 今日使用次数 | 当日累计使用次数 | 红外传感器 |
| 间隔时长 | 距上次使用时间 | 系统计算 |
| 清洁度 | 当前清洁程度百分比 | 传感器检测 |

**智能猫垫显示指标：**

| 指标 | 说明 | 数据来源 |
|------|------|----------|
| 剩余量 | 猫垫耗材剩余百分比 | 称重传感器 |
| 预计可用天数 | 按消耗速度预估 | 算法计算 |
| 状态 | 正常/需更换 | 系统判断 |

### 3.2 智能猫厕所模块

#### 3.2.1 使用数据统计

| 功能项 | 规格 |
|--------|------|
| **今日使用次数** | 实时统计，数字展示 |
| **平均如厕时长** | 今日所有使用的平均时长 |
| **当前清洁度** | 百分比显示，低于70%提醒清洁 |
| **上次使用间隔** | 时间差显示 |

#### 3.2.2 使用记录时间线

| 属性 | 规格 |
|------|------|
| **展示形式** | 时间线列表 |
| **单条记录内容** | 时间、猫咪（多猫识别）、时长、状态 |
| **状态判断** | 正常/偏短/偏长/异常 |
| **历史范围** | 近30天可查询 |
| **筛选功能** | 按日期、按猫咪筛选 |

#### 3.2.3 远程清洁控制

| 属性 | 规格 |
|------|------|
| **触发方式** | APP按钮点击 |
| **前置条件** | 设备在线、无人/宠物在设备内 |
| **安全检测** | 红外检测有生命体则禁止启动 |
| **执行反馈** | 清洁中/清洁完成/清洁失败 |
| **清洁耗时** | 约60-90秒 |

#### 3.2.4 智能模式设置

| 模式 | 说明 |
|------|------|
| **自动清洁** | 检测到使用后自动启动清洁 |
| **定时清洁** | 设置固定时间自动清洁 |
| **手动清洁** | 仅通过APP/按钮触发 |
| **静音时段** | 指定时间段禁止自动清洁 |

#### 3.3.1 剩余量监控

| 属性 | 规格 |
|------|------|
| **展示形式** | 环形进度图 + 百分比数字 |
| **更新频率** | 每小时自动更新 |
| **精度要求** | ±5% |

| 属性 | 规格 |
|------|------|

| **数据内容** | 每日消耗百分比 |

| 触发条件 | 提醒方式 |
|----------|----------|

### 3.4 健康中心模块

#### 3.4.1 健康评分系统

**评分维度：**

| 维度 | 权重 | 数据来源 | 判断标准 |
|------|------|----------|----------|
| 如厕频率 | 30% | 猫厕所使用记录 | 与历史平均值对比 |
| 如厕时长 | 25% | 猫厕所使用记录 | 正常范围2-5分钟 |
| 活动量 | 25% | 猫垫使用记录 | 与历史平均值对比 |
| 休息质量 | 20% | 猫垫压力传感 | 休息时长规律性 |

**评分等级：**

| 分数范围 | 等级 | 颜色 |
|----------|------|------|
| 90-100 | 优秀 | 绿色 |
| 70-89 | 良好 | 绿色 |
| 50-69 | 一般 | 黄色 |
| 30-49 | 需关注 | 橙色 |
| 0-29 | 需就医 | 红色 |

#### 3.4.2 异常预警机制

| 异常类型 | 触发条件 | 预警级别 |
|----------|----------|----------|
| 如厕过频 | 24小时内超过10次 | ⚠️ 关注 |
| 如厕过少 | 24小时内少于1次 | ⚠️ 关注 |
| 如厕时长异常 | 单次超过10分钟或少于30秒 | ⚠️ 关注 |
| 连续异常 | 连续3天数据异常 | 🚨 警告 |
| 多项异常 | 两个以上指标同时异常 | 🚨 警告 |

#### 3.4.3 健康报告

| 属性 | 规格 |
|------|------|
| **报告周期** | 周报/月报 |
| **报告内容** | 数据总结、趋势分析、健康建议 |
| **生成时间** | 每周一/每月1日自动生成 |
| **分享功能** | 支持分享给兽医 |

## 四、非功能性需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| APP启动时间 | < 3秒 |
| 页面加载时间 | < 2秒 |
| 设备控制响应 | < 1秒 |
| 数据刷新延迟 | < 5秒 |

### 4.2 安全要求

| 要求 | 说明 |
|------|------|
| 数据传输 | HTTPS加密传输 |
| 用户认证 | JWT Token + 刷新机制 |
| 密码存储 | 加盐哈希存储 |
| 设备认证 | 一机一密 + 动态密钥 |

### 4.3 兼容性要求

| 平台 | 版本要求 |
|------|----------|
| iOS | iOS 13.0及以上 |
| Android | Android 8.0及以上 |
| 微信小程序 | 基础库2.20.0及以上 |

### 4.4 可用性要求

| 指标 | 要求 |
|------|------|
| 系统可用性 | 99.9% |
| 故障恢复时间 | < 30分钟 |
| 数据备份 | 每日备份，保留30天 |

## 五、用户故事

### 5.1 核心用户故事

**US-001：设备绑定**
> 作为一名新用户，我希望能够快速绑定设备，以便开始使用APP管理我的智能宠物用品。

**验收标准：**
- [ ] 支持扫码绑定
- [ ] 支持手动输入序列号绑定
- [ ] 绑定过程有清晰的引导
- [ ] 绑定成功后自动跳转设备详情

**US-002：查看猫厕所使用情况**
> 作为一名宠物主人，我希望能够查看猫咪的如厕记录，以便了解它的健康状况。

**验收标准：**
- [ ] 首页显示今日使用次数
- [ ] 详情页显示完整使用记录
- [ ] 支持查看历史记录（近30天）
- [ ] 异常使用有标记提示

**US-003：猫垫剩余量提醒**
> 作为一名宠物主人，我希望在猫垫耗材快用完时收到提醒，以便及时购买更换。

**验收标准：**
- [ ] 剩余量低于30%时推送提醒
- [ ] 提醒消息包含预计可用天数
- [ ] 支持设置提醒阈值
- [ ] 支持一键跳转购买页面

**US-004：远程启动清洁**
> 作为一名上班族，我希望能够远程控制猫厕所清洁，以便保持厕所卫生。

**验收标准：**
- [ ] 支持一键启动清洁
- [ ] 有安全检测机制（检测是否有宠物在内）
- [ ] 显示清洁进度和结果
- [ ] 清洁完成有通知

**US-005：健康异常预警**
> 作为一名关注宠物健康的主人，我希望在猫咪行为异常时收到预警，以便及时关注或就医。

**验收标准：**
- [ ] 系统自动检测异常行为
- [ ] 异常情况及时推送通知
- [ ] 提供异常说明和建议
- [ ] 支持分享报告给兽医
