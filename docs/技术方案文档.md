| **对象存储** | 阿里云OSS / MinIO | 图片、文件存储 |
| **推送服务** | 极光推送 / 阿里云推送 | APP消息推送 |

## 三、通信协议

### 3.1 MQTT Topic 设计

| Topic | 方向 | 说明 |
|-------|------|------|
| `device/{device_id}/status` | 设备→云端 | 设备状态上报 |
| `device/{device_id}/data` | 设备→云端 | 传感器数据上报 |
| `device/{device_id}/command` | 云端→设备 | 控制指令下发 |
| `device/{device_id}/ota` | 云端→设备 | OTA升级通知 |
| `user/{user_id}/notification` | 云端→APP | 消息推送 |

### 3.2 设备数据上报格式

```json
{
    "device_id": "MT2025010001",
    "device_type": "cat_toilet",
    "timestamp": 1704067200000,
    "data": {
        "weight": 4.5,           // 猫咪重量(kg)
        "duration": 180,         // 使用时长(秒)
        "event_type": "usage",   // 事件类型
        "cleanliness": 85,       // 清洁度(%)
        "battery": 100,          // 电量(%)
        "temperature": 25.5,     // 温度(℃)
        "humidity": 60           // 湿度(%)
    }
}
```

### 3.3 控制指令格式

```json
{
    "command_id": "CMD202501010001",
    "command_type": "clean",
    "timestamp": 1704067200000,
    "params": {
        "mode": "auto",
        "duration": 60
    }
}
```

### 3.4 指令响应格式

```json
{
    "command_id": "CMD202501010001",
    "status": "success",
    "timestamp": 1704067201000,
    "message": "清洁完成"
}
```

## 四、API 接口设计

### 4.1 RESTful API 规范

**基础路径：** `https://api.meodou.com/v1`

**请求头：**

```
Content-Type: application/json
Authorization: Bearer {access_token}
X-Device-ID: {device_id}
X-Request-ID: {uuid}
```

**响应格式：**

```json
{
    "code": 0,
    "message": "success",
    "data": { ... },
    "timestamp": 1704067200000
}
```

### 4.2 核心接口列表

#### 用户模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 手机号登录 | POST | /auth/login | 短信验证码登录 |
| 微信登录 | POST | /auth/wechat | 微信授权登录 |
| 获取用户信息 | GET | /user/profile | 获取用户详情 |
| 更新用户信息 | PUT | /user/profile | 更新用户信息 |

#### 设备模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取设备列表 | GET | /devices | 获取用户设备列表 |
| 绑定设备 | POST | /devices/bind | 绑定新设备 |
| 解绑设备 | DELETE | /devices/{id} | 解绑设备 |
| 获取设备详情 | GET | /devices/{id} | 获取设备状态 |
| 获取设备数据 | GET | /devices/{id}/data | 获取传感器数据 |
| 发送控制指令 | POST | /devices/{id}/command | 远程控制设备 |

#### 健康模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取健康评分 | GET | /health/score | 获取宠物健康评分 |
| 获取健康报告 | GET | /health/report | 获取健康报告 |
| 获取使用记录 | GET | /health/records | 获取使用记录 |

## 六、安全方案

### 6.1 认证授权

```
┌─────────┐                    ┌─────────┐                    ┌─────────┐
│  客户端  │                    │ API网关  │                    │ 用户服务 │
└────┬────┘                    └────┬────┘                    └────┬────┘
     │                              │                              │
     │  1. 登录请求                  │                              │
     │ ─────────────────────────────►│                              │
     │                              │  2. 转发登录请求               │
     │                              │ ─────────────────────────────►│
     │                              │                              │
     │                              │  3. 返回access_token          │
     │                              │    + refresh_token            │
     │                              │ ◄─────────────────────────────│
     │  4. 返回token                 │                              │
     │ ◄─────────────────────────────│                              │
     │                              │                              │
     │  5. 业务请求 + access_token   │                              │
     │ ─────────────────────────────►│                              │
     │                              │  6. 验证token                 │
     │                              │ ─────────────────────────────►│
     │                              │  7. 返回用户信息               │
     │                              │ ◄─────────────────────────────│
     │                              │                              │
     │  8. 返回业务数据              │                              │
     │ ◄─────────────────────────────│                              │
     │                              │                              │
```

### 6.2 Token 设计

**Access Token (有效期: 2小时)**

```json
{
    "sub": "10001",
    "name": "用户昵称",
    "phone": "138****8888",
    "iat": 1704067200,
    "exp": 1704074400
}
```

**Refresh Token (有效期: 7天)**

```json
{
    "sub": "10001",
    "type": "refresh",
    "iat": 1704067200,
    "exp": 1704672000
}
```

### 6.3 设备安全

| 安全措施 | 说明 |
|----------|------|
| 一机一密 | 每台设备有唯一密钥，出厂预烧录 |
| TLS加密 | MQTT over TLS，数据传输加密 |
| 动态Token | 设备Token定期刷新 |
| 固件签名 | OTA固件需签名验证 |
